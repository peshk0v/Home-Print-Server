Домашний принт-сервер на Raspberry Pi

Полное руководство: превратите старый USB-принтер в современное сетевое устройство.
Установка одной командой, минимальные усилия.

Этот гайд проведёт вас через каждый этап: от подготовки SD-карты до печати с любого устройства в вашей сети. Всё максимально просто и структурировано.

Что вам понадобится

КомпонентРекомендации
Raspberry PiЛюбая модель с Wi-Fi (3B, 4B, Zero 2 W)
microSD-карта8–32 ГБ, класс 10
Блок питания5В / 2.5–3А (официальный или качественный)
USB-принтерЛюбой рабочий принтер с USB-портом
КомпьютерДля подготовки карты (Windows / macOS / Linux)

Шаг 1: Готовим загрузочную флешку с Raspberry Pi OS Lite

  • Скачайте Raspberry Pi Imager с официального сайта (raspberrypi.com/software) и установите.
  • Запустите Imager, выберите вашу модель Raspberry Pi (например, Raspberry Pi 4).
  • Выберите ОС → Raspberry Pi OS (Lite, 64-bit) (или 32-битную, если ваша модель не поддерживает 64).
  • Нажмите на шестерёнку (настройки ОС). Это важно для «безголовой» установки:
    • Имя хоста: printserver
    • Включить SSH → «Использовать пароль для аутентификации»
    • Пользователь: pi (придумайте надёжный пароль)
    • Настроить Wi-Fi: укажите SSID и пароль вашей сети
  • Выберите накопитель (вашу microSD-карту) и нажмите «Записать». Дождитесь окончания.
Совет: Если вы не укажете Wi-Fi сейчас, подключить Pi к сети позже будет сложнее. Рекомендуем заполнить всё на этом этапе.

Шаг 2: Подключаем Raspberry Pi

  • Подключите USB-принтер к Raspberry Pi.
  • Вставьте microSD-карту в Raspberry Pi.
  • Подключите блок питания — плата начнёт загрузку.
Подождите 1–2 минуты для первой загрузки и подключения к Wi-Fi.

Шаг 3: Подключаемся через SSH

Найдите IP-адрес вашего Raspberry Pi (посмотрите в веб-интерфейсе роутера или используйте программу сканирования сети).

Откройте терминал (командную строку) и выполните:

ssh pi@printserver.local

Если имя не работает — используйте прямой IP:

ssh pi@192.168.x.x

При первом подключении введите yes, затем пароль пользователя pi (символы не отображаются).

Если подключение не удаётся: убедитесь, что вы включили SSH в настройках Imager, и проверьте правильность Wi-Fi данных.

Шаг 4: Автоматическая установка (одна команда)

Теперь, когда вы подключены к Raspberry Pi по SSH, выполните команду ниже. Скрипт сам установит всё необходимое.

curl -sSL https://raw.githubusercontent.com/peshk0v/Home-Print-Server/refs/heads/main/installer.sh | sudo bash

Что сделает скрипт: обновит систему, установит CUPS, Samba, Avahi, настроит удалённый доступ, добавит драйверы и перезапустит службы. Процесс займёт 5–15 минут.

Не закрывайте терминал и не выключайте питание до появления сообщения об успешном завершении.

Шаг 5: Добавляем принтер через веб-интерфейс CUPS

  • Откройте браузер на любом устройстве в той же сети.
  • Перейдите по адресу: http://[IP-адрес вашего Raspberry Pi]:631 (например http://192.168.1.100:631).
  • Нажмите вкладку Administration → Add Printer.
  • Введите логин pi и ваш пароль.
  • Выберите свой USB-принтер из списка "Local Printers".
  • Поставьте галочку Share This Printer (общий доступ).
  • Выберите драйвер (CUPS предложит подходящий; при необходимости — установите драйвер из списка или выберите "Raw").
  • Нажмите Add Printer.
Если принтер не отображается: выполните в SSH команду lsusb — он должен быть в списке устройств. После этого перезагрузите Pi (sudo reboot).

Шаг 6: Печатаем с любых устройств

После добавления принтер автоматически станет доступен в сети.

Windows Панель управления → Устройства и принтеры → Добавить принтер → автоматическое обнаружение.
macOS / iOS Принтер появится в списке автоматически (Bonjour).
Linux Обычно находится через настройки печати (CUPS/Avahi).
Android Используйте Mopria Print Service или PrintHand.

Если автоматическое обнаружение не работает, добавьте принтер вручную по адресу:
ipp://[IP-адрес_Pi]:631/printers/[Имя_принтера]

Как это работает (кратко)

CUPS — сервер печати: принимает задания, управляет очередью, связывается с принтером.
Samba — делает принтер видимым для Windows-устройств.
Avahi — обеспечивает Zero‑конфигурацию (Bonjour), чтобы принтер находился автоматически.

Вместе эти три компонента превращают Raspberry Pi в полноценный сетевой принт-сервер, совместимый с любыми современными ОС.

Возможные проблемы и их решение

ПроблемаРешение
Raspberry Pi не виден в сетиПроверьте правильность Wi-Fi данных в Imager. Подождите 2 минуты. Посмотрите список устройств в роутере.
Ошибка SSH "Connection refused"Убедитесь, что включили SSH в настройках Imager перед записью. Перезапишите карту.
Принтер не появляется в CUPSВыполните lsusb в SSH — принтер должен быть в списке. Перезагрузите Pi командой sudo reboot.
Задание в очереди, но печать не идётПроблема с драйвером. В веб-интерфейсе CUPS выберите другой драйвер или отправьте тестовую страницу.

Поздравляем!

Вы успешно превратили Raspberry Pi в современный домашний принт-сервер. Теперь ваш старый USB-принтер доступен для всех устройств в сети: ноутбуков, смартфонов, планшетов.
